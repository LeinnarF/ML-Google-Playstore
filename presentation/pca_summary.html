📊 Principal Component Analysis (PCA)

Executive Summary - Dimensionality Reduction & App Quality Analysis

🎯 Analysis Overview

Python - Setup & Data Loading 
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
import numpy as np

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

df = pd.read_csv('dataset/googleplaystore_cleaned.csv')

🎯 Objectives

  • Reduce high-dimensional app features to principal components
  • Identify patterns that explain app quality and success
  • Visualize apps in 2D/3D space based on their characteristics
  • Find top-performing apps across Free and Paid categories
  • Understand which features contribute most to app quality

🔧 Data Preprocessing

1. Handling "Varies with Device"

Python 
# Handle 'Varies with device' in 'Size' column
df['Size_varies'] = (df['Size'] == 'Varies with device').astype(int)
df['Size'] = pd.to_numeric(df['Size'], errors='coerce')

# Fill based on the median size of apps in the same category
df['Size'] = df.groupby('Category')['Size'].transform(
    lambda x: x.fillna(x.median())
)

2. Feature Engineering

Python 
# Log transformations for skewed features
df['Log_Installs'] = np.log1p(df['Installs'] + 1)
df['Log_Reviews'] = np.log1p(df['Reviews'] + 1)
df['Log_Size'] = np.log1p(df['Size'] + 1)

# Avoid division by zero
df['Rating'] = df['Rating'].replace(0, 1e-15)

# Derived features
df['Premium_score'] = df['Price'] / df['Rating']
df['Review_ratio'] = df['Reviews'] / (df['Installs'] + 1)
df['Engagement'] = df['Rating'] * np.log1p(df['Reviews'])

📐 PCA Implementation

Feature Selection & Scaling

Python 
# Select numerical features for PCA
features = [
    'Rating', 'Log_Reviews', 'Log_Size', 'Log_Installs',
    'Price', 'Premium_score', 'Review_ratio', 'Engagement'
]

X = df[features].fillna(0)

# Standardize features (mean=0, std=1)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Apply PCA
pca = PCA(n_components=4)
pca_result = pca.fit_transform(X_scaled)

# Explained variance
explained_variance = pca.explained_variance_ratio_
print(f'Variance explained: {explained_variance}')

📊 PCA Results

  • PC1: Captures app popularity & engagement (highest variance)
  • PC2: Represents monetization strategy (Free vs Paid)
  • PC3: Size and technical requirements
  • PC4: User engagement patterns
  • Total Components: 4 principal components used

📈 Visualization & Insights

2D Scatter Plot by App Type

Python 
# Create DataFrame with PCA results
pca_df = pd.DataFrame({
    'PC1': pca_result[:, 0],
    'PC2': pca_result[:, 1],
    'PC3': pca_result[:, 2],
    'PC4': pca_result[:, 3],
    'App_type': df['Type'],
    'App': df['App']
})

# Visualize
plt.figure(figsize=(10, 6))
sns.scatterplot(
    x='PC1', y='PC2',
    hue='App_type',
    data=pca_df,
    palette={'Free': 'blue', 'Paid': 'red'},
    alpha=0.6
)
plt.title('PCA: Apps by Type (PC1 vs PC2)')
plt.xlabel('PC1 - App Quality & Popularity')
plt.ylabel('PC2 - Monetization Strategy')
plt.show()

PCA 2D Visualization: Apps colored by Type (Free vs Paid)
