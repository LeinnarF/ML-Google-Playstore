6️⃣ Installs - Format Normalization

Issue: Formatted as '10,000+', '500,000+' Solution: Removed '+' and commas, converted to int64

Range: 0 to 1,000,000,000+ installs

Python 
# Replace edge case
replace = {'0+': '1+'}
df['Installs'] = df['Installs'].replace(replace)

# Remove '+' and ',' then convert to numeric
df['Installs'] = df['Installs'].str.replace('+', '').str.replace(',', '')
df['Installs'] = pd.to_numeric(df['Installs'], errors='coerce')

7️⃣ Size - Unit Standardization

Issue: Mixed units (M, k, MB, KB) + 1,227 "Varies" Solution: Normalized all to KB

Special handling: "Varies with device" retained as-is

Python 
# Standardize 'Size' column
def convert_size(size):
    if size == 'Varies with device':
        return 'Varies with device'
    elif size.endswith('M'):
        return float(size[:-1]) * 1024  # Convert MB to KB
    elif size.endswith('k'):
        return float(size[:-1])  # Already in KB
    else:
        return np.nan  # Handle unexpected formats
    
df['Size'] = df['Size'].apply(convert_size)

8️⃣ Type - Consistency Resolution

Issue: 1 missing + inconsistencies with Price Solution: Inferred from Price (>0 = Paid, =0 = Free)
Python 
# Consistency check for 'Type' column
inconsistent_type = df[(df['Price'] > 0) & (df['Type'] != 'Paid') | 
                       (df['Price'] == 0) & (df['Type'] != 'Free')]

# Resolve NaN values in 'Type' column based on 'Price'
df.loc[df['Price'] > 0, 'Type'] = 'Paid'
df.loc[df['Price'] == 0, 'Type'] = 'Free'

9️⃣ Content Rating - Category Consolidation

Issue: Rare categories "Adults only 18+" & "Unrated" Solution: Merged into existing categories

Mapping: "Adults only 18+" → "Mature 17+" | "Unrated" → "Everyone"

Python
